Admin / setup

EastMallBuy katalógus kezelői útmutató

Ez az oldal a beüzemeléshez, JSON fájlok kezeléséhez, affiliate linkek ellenőrzéséhez és hibakereséshez készült. A vásárlói felület külön oldalon fut.

Publikus katalógus megnyitása Diagnosztikai eszközök

A) Alkalmazás áttekintése

Ez az app EastMallBuy export JSON fájlokból épít statikus, kereshető termékkatalógust. A publikus oldalon a látogatók termékeket böngésznek, az admin oldalon pedig te tudod ellenőrizni a betöltést, a manifestet és a fájlokat.

A működés röviden: a böngésző betölti a data/manifest.json fájlt, végigolvassa a benne felsorolt exportokat, normalizálja a termékeket, deduplikál, majd kirajzolja a katalógust.

B) Első beüzemelés

A teljes projektmappát tedd webszerver alá. XAMPP esetén például: c:\xampp\htdocs\embimport, majd nyisd meg: http://localhost/embimport/.

A file:// megnyitás nem ajánlott, mert a böngésző blokkolhatja a fetch() alapú manifest és JSON betöltést. Használj XAMPP-ot, localhostot vagy bármilyen statikus tárhelyet. 

/index.html
/admin.html
/assets/css/style.css
/assets/js/public-app.js
/assets/js/admin-app.js

/data/zokni/shop1.json

C) JSON fájlok kezelése

Az exportokat kategóriamappákba érdemes tenni. Például a cipős fájlok menjenek a data/cipo/ mappába, a zoknik a data/zokni/ mappába.

Manifest példa:

{
  "datasets": [
    {
      "path": "data/cipo/shop1.json",
      "category": "Cipő",
      "label": "Cipő / Shop 1"
    }
  ]
}

Minimum elvárt JSON szerkezet:

{
  "items": [
    {
      "itemId": "123",
      "title": "Példa termék",
      "price": 12.5,
      "url": "https://eastmallbuy.com/index/item/index.html?tp=micro&tid=123",
      "affiliateUrl": "https://eastmallbuy.com/index/item/index.html?tp=micro&tid=123&inviter=gelenfarkas"
    }
  ]
}

D) Kézi import

Kézi importot akkor használj, ha gyorsan szeretnél több JSON-t ellenőrizni, vagy file:// módban nyitottad meg az oldalt, ahol a manifestes betöltés blokkolódhat. Választhatsz több JSON fájlt, vagy támogatott böngészőben komplett mappát is.

E) Affiliate link működés

A publikus katalógus minden kattintható terméklinknél affiliate linket használ. Ha az exportban van affiliateUrl, mindig az kerül ki. Ha nincs, az app újragenerálja a linket az itemId, tp és inviter alapján.

Ellenőrzés: a publikus termékkártya gombjának URL-jében szerepelnie kell az inviter=gelenfarkas paraméternek.

F) Hibakeresés / debug

Ha nincs termék, ezeket nézd meg sorrendben:

  • localhost alatt fut-e az oldal, nem file:// módban?
  • elérhető-e a data/manifest.json?
  • a manifestben szereplő path tényleg létező JSON fájlra mutat?
  • van-e items tömb a JSON-ban?
  • UTF-8 kódolásúak-e a fájlok?
  • a debug panel mutat-e fetch, parse vagy normalizálási hibát?

G) Gyors ellenőrző lista

  • Manifest létezik és érvényes JSON.
  • Dataset path jó és HTTP-n elérhető.
  • Az exportban van items tömb.
  • Az affiliate link tartalmaz inviter paramétert.
  • A fájlok UTF-8 kódolásúak.
  • A publikus oldalon megjelennek a termékek.
