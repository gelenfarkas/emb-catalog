EastMallBuy katalógus kezelői útmutató

Naprakész diagnosztikai oldal a manifest alapú JSON katalógushoz, multi-category tagekhez, affiliate linkekhez és cache-biztos frissítésekhez.

Publikus katalógus megnyitása Diagnosztikai eszközök

Alkalmazás áttekintése

Az app frontend-only EastMallBuy affiliate katalógus. A publikus oldal a manifestben felsorolt JSON exportokat tölti be, normalizálja, deduplikálja, majd kereshető és szűrhető termékkártyákat renderel.

Folyamat: manifest.json → dataset JSON → normalizálás → multi-tag kategorizálás → deduplikáció → szűrés/rendezés → affiliate linkes kártyák.

Első beüzemelés

HTTP-n keresztül futtasd. XAMPP esetén: c:\xampp\htdocs\embimport, majd http://localhost/embimport/. Python szerverrel: python -m http.server.

A file:// mód nem megbízható, mert a böngésző blokkolhatja a fetch() alapú manifest és JSON betöltést.

Manifest / JSON kezelés

A data/manifest.json datasets tömbje mondja meg, mely exportokat kell betölteni. A category és label a dataset azonosítását segíti, a terméktageket a title alapú felismerés is bővíti.

A manifest kategória manuális hozzárendelésként elsőbbséget kap, az automatikus kategóriák pedig deduplikált kiegészítő címkeként jelennek meg. 

{
  "datasets": [
    {
      "path": "data/cipo/ohero-export.json",
      "category": "Cipő",
      "label": "Cipő / OHERO STUDIO"
    }
  ]
}
{
  "items": [
    {
      "itemId": "123",
      "title": "Outdoor running shoes",
      "price": 12.5,
      "url": "https://eastmallbuy.com/index/item/index.html?tp=micro&tid=123",
      "affiliateUrl": "https://eastmallbuy.com/index/item/index.html?tp=micro&tid=123&inviter=gelenfarkas",
      "sellerName": "OHERO STUDIO"
    }
  ]
}

Kategorizálás

A CATEGORY_MAP HU, EN és CN kulcsszavakat tartalmaz. A detectCategories(title) minden kategóriára score-t számol, és minden találatot hozzáad a termékhez.

A legjobb találat a primary kategória: product.category, product.categoryId és product.categoryLabel. Az összes tag a product.categories tömbben van.

Multi-tag rendszer

Egy termék több kategóriába tartozhat. Például egy outdoor futócipő egyszerre lehet Cipő, Sport és Outdoor.

A kategóriaszűrés a product.categories.includes(selectedCategory) logikát használja, ezért másodlagos tag alatt is megjelenik a termék.

Keresés

A kereső normalizált szöveget használ: cím, TID, seller, forrás, dataset label és kategóriatag is bekerül a keresési indexbe. Kategória jellegű keresésnél a kapcsolódó HU, EN és CN kulcsszavak is segítik a találatot.

Affiliate linkek

A publikus kártyák affiliate URL-t használnak. Ha van affiliateUrl, az kerül ki; ha nincs, az app generál linket az itemId, tp és inviter alapján.

A Link másolása gomb ugyanazt az affiliate linket másolja. Fallbackként generált affiliate linket vagy eredeti termék URL-t használ.

Cache / frissítés

A projekt tudatosan nem használ kliensoldali cache-t. A központi verzióforrás az assets/js/app-version.js, és minden kritikus erőforrás verziózott URL-t kap: bootstrap JS, CSS, fő app JS, favicon, manifest és dataset JSON.

Az Apache konfiguráció és a PHP fallback no-store headereket küld. Nincs service worker, PWA cache, localStorage/sessionStorage alapú JSON cache, polling vagy auto-refresh. Új deploy után azonnali frissülés a cél, Chrome-ban is.

Hibakeresés

  • HTTP-n fut az oldal, nem file:// alatt.
  • A data/manifest.json?v=... elérhető és valid JSON.
  • A manifest path mezői létező dataset fájlokra mutatnak.
  • A dataset tartalmaz items tömböt.
  • Új deploy után változott az APP_VERSION.
  • Chrome DevTools Network fülön friss ?v= értékek látszanak.
  • A cache-utils.js és app-version.js is verziózott URL-ről tölt.
  • A debug panel nem mutat fetch, parse vagy normalizálási hibát.

Deploy / frissítés

  • Emeld az APP_VERSION értékét.
  • Töltsd fel a HTML, JS, CSS, JSON, SVG és PHP fájlokat.
  • Ellenőrizd az admin debug panelt.
  • Nézd meg, hogy a publikus oldal friss verziós asseteket tölt.
  • Normál Chrome reload után is az új buildnek kell megjelennie.
